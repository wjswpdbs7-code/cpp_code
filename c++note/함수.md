## 함수

#### 사용자 함수
예시) void func(){} 

#### 표준함수
C++에서 제공하는 함수. 예시) cout, cin, getline, strcmp 등

#### 메인함수 
프로그램의 시작점이 되는 함수. 반드시 하나만 존재해야 하며, int main() 형태로 정의됨.

#### 멤버함수
클래스의 멤버로 정의된 함수. 객체가 있어야 호출 가능. 예시) string 클래스의 length() 함수, vector 클래스의 size() 함수 등

#### 함수 구조

반환값 함수명(인자) {
    // 함수가 수행할 작업
    return 반환값;(있는경우)
}
#### 반환값
함수가 작업을 수행한 후 결과로 돌려주는 값의 자료형. 예시) int, void, double 등

void가 아니면 반드시 return문이 있어야 함. void는 반환값이 없음을 의미.

#### 인자

함수에 전달되는 값이나 변수. 함수가 작업을 수행하는 데 필요한 정보를 제공. 예시) int a, double b, string s 등
실제 호출될때 변수명은 달라도 상관없음. 
간단한 함수와 호출예시
int add(int x, int y) {
    return x + y;
}
int main() {
    int result = add(3, 5);
    cout << result << endl; // 8
    return 0;
}

#### 값에 의한 호출
###### 함수에 인자를 전달할 때, 인자의 값을 복사해서 전달하는 방식. 함수 내부에서 인자의 값을 변경해도, 원래의 값에는 영향을 미치지 않음.

###### 특징
1. 원본 유지
2. 생성자 및 소멸자 비대칭 호출 
3. 사용이 쉽다

include <iostream>
using namespace std;

void swap(int a, int b){
    int temp = a;
    a = b;
    b = temp;
}

int main(){
    int m = 2 , n = 9;
    swap(m,n);
    cout << m << " " << n << endl;
    return 0;
}
###### !!swap함수는 이 코드에서 역할 수행을 못함. m과 n의 값이 변경되지 않고 그대로 출력됨 (2 9)

#### 현재 함수의 스택 메모리 그림
원본 m과 n이 각각 100번지와 200번지에 저장되어 있다고 가정
-------------------------------------------------
500번지|  temp  |  2
-------------------------------------------------
400번지|   b    |  9
-------------------------------------------------
300번지|   a    |  2
-------------------------------------------------
200번지|   n    |   9
-------------------------------------------------
100번지|   m    |   2
------------------------------------------------

1. swap함수에서 a와 b는 m과 n의 값을 복사해서 받음 (a = 2, b = 9)
2. temp에 a의 값이 저장됨 (temp = 2)
3. a의 값이 b가 가리키는 값으로 변경 (a = 9)
4. b의 값이 temp가 가리키는 값으로 변경 (b = 2)
5. swap함수가 스택 영역에서 내려감
6. 메인함수로 돌아감
7. m과 n의 값은 변경되지 않고 그대로 출력됨 (2 9)   







#### 주소에 의한 호출
###### 함수에 인자를 전달할 때, 인자의 주소를 전달하는 방식. 함수 내부에서 인자의 값을 변경하면, 원래의 값도 변경됨.

###### 특징
1. 원본 변경
2. 객체를 생성하지 않음(생성자 소멸자 상관 없음)
3. 포인터를 사용하기에 사용법이 어렵다

include <iostream>
using namespace std;

void swap(int *a, int *b){
    int temp = *a;
    *a = *b;
    *b = temp;
}

int main(){
    int m = 2 , n = 9;
    swap(&m, &n);
    cout << m << " " << n << endl;
    return 0;
}


-----------------------------------------------
500번지|  temp  |  2-> 끝나면 지워짐
-----------------------------------------------
400번지|   *b    |  200번지-> 끝나면 지워짐
------------------------------------------------
300번지|   *a    |  100번지-> 끝나면 지워짐
------------------------------------------------
200번지|   n    |   9
------------------------------------------------
100번지|   m    |   2 
------------------------------------------------

순서
1. swap함수에서 a와 b는 m과 n의 주소를 받음 (a = 100번지, b = 200번지)
2. temp에 a가 가리키는 값이 저장됨 (temp = 2)
3. a가 가리키는 값이 b가 가리키는 값으로 변경 (m = 9)
4. b가 가리키는 값이 temp가 가리키는 값으로 변경 (n = 2)
5. swap함수가 스택 영역에서 내려감
6. 메인함수로 돌아감
7. m과 n의 값이 변경되어 출력됨 (9 2)

## 참조자에 의한 호출

###### 특징
1. 원본 변경
2. 객체를 생성하지 않음(생성자 소멸자 상관 없음)
3. 포인터보다 사용법이 간단하다

## 왜 int num을 하면 값이 그대로 12가 나오고 int &num을 하면 13이 나오는가?
int num은 num이 함수로 전달될 때, num의 값이 복사되어 함수 내부에서 num이라는 새로운 변수가 생성되고, 그 변수에 복사된 값이 저장됩니다. 
따라서 함수 내부에서 num의 값을 변경해도, main 함수에서 선언된 n 변수의 값은 변경되지 않습니다.


반면에 int &num은 num이 함수로 전달될 때, num의 참조자가 전달되어 함수 내부에서 num이라는 새로운 변수가 생성되지 않고, main 함수에서 선언된 n 변수의 참조자가 num에 할당됩니다.
선언된 n 변수의 참조자가 num에 할당 된다는 말은 num이 n을 가리키게 된다는 뜻입니다. 

------------------------------------------------
500번지 함수 내부|  num  |  12->13
------------------------------------------------
200번지 main 함수|   n    |   12->13
------------------------------------------------


int &num의 의미를 간단히 설명하면 num은 n의 별명입니다. num을 통해 n에 접근할 수 있고, num의 값을 변경하면 n의 값도 변경됩니다.
따라서 함수 내부에서 num의 값을 변경하면, main 함수에서 선언된 n 변수의 값도 변경됩니다.

호출할때 plus1(n)가 뜻하는것을 풀어 말하면 
plus1(n)는 plus1 함수에 n이라는 변수를 전달하는 것을 의미합니다. 
plus1 함수는 int &num이라는 매개변수를 가지고 있기 때문에, n 변수의 참조자가 num에 할당됩니다.
n변수의 참조자란 n이 저장된 메모리 주소를 가리키는 포인터와 유사한 개념입니다.
plus1(n)의 실행과정은 다음과같음
1. plus1 함수가 호출되면, n 변수의 참조자가 num에 할당됩니다. 즉, num은 n을 가리키게 됩니다.
2. plus1 함수 내부에서 num의 값을 변경하면, num이 n을 가리키고 있기 때문에, n의 값도 변경됩니다.
3. plus1 함수가 종료되면, num은 스택에서 제거되고, n 변수는 main 함수에서 계속 존재합니다.

코드를 주소 값을 호출하는 방식으로 변경하면
void plus1(int *num);// 함수 선언 선언시에는 *으로하고 호출시에는 &으로 하는 이유는 plus1 함수가 int *num이라는 매개변수를 가지고 있기 때문입니다.
                       //여기서는 num이라는 포인터 변수를 선언한것이고 호출할때는 n 변수의 주소를 전달해야 하기 때문입니다.

int main(){
    int n = 12;
    plus1(&n);// 함수 호출 이때는 &n이 뜻하는것을 풀어 말하면 plus1 함수에 n 변수의 주소를 전달하는 것을 의미합니다. 
              //plus1 함수는 int *num이라는 매개변수를 가지고 있기 때문에, n 변수의 주소가 num에 할당됩니다.
    cout << n << endl; 
}
void plus1(int *num){//함수 정의 여기서의 int *num은 포인터 선언
    *num = *num + 1;// 여기서의 *num은 num이 가리키는 주소에 저장된 값을 의미 포인터를 사용하면 메인의 값이변하고 아닐경우에는 안변하는 이유는 포인터는 주소값을 전달하기 때문에 함수 내부에서 num이 가리키는 주소에 저장된 값을 변경하면, main 함수에서 선언된 n 변수의 값도 변경됩니다. 반면에 int num은 값이 복사되어 전달되기 때문에, 함수 내부에서 num의 값을 변경해도, main 함수에서 선언된 n 변수의 값은 변경되지 않습니다.
}


메모리 구조로 설명
------------------------------------------------
500번지 함수 내부|  *num(100번지)  |  12->13 
------------------------------------------------
200번지 main 함수|   n(100번지)    |   12->13 
------------------------------------------------
100번지 main 함수|   n    |   12->13 -
------------------------------------------------
함수 내에서 진행된것은 리턴값이 없으면 함수가 종료되면 스택에서 제거됨.
그런데 코드의 결과값이 13이 나오는 이유는 plus1 함수에서 num이 n의 주소를 가리키고 있기 때문에, *num을 통해 n의 값을 변경할 수 있기 때문입니다.
plus1 함수에서 *num = *num + 1;은 num이 가리키는 주소에 저장된 값을 1 증가시키는 것을 의미합니다.
