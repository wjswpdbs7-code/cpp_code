## string 함수 설명

string은 C++에서 문자열을 다루기 위한 클래스입니다. string 클래스는 문자열을 저장하고 조작하는 다양한 기능을 제공합니다. string 클래스는 동적 메모리를 사용하여 문자열을 관리하므로, 문자열의 길이에 따라 자동으로 메모리를 할당하고 해제할 수 있습니다. 또한, string 클래스는 문자열 연결, 비교, 검색 등의 다양한 연산을 지원하여 문자열을 쉽게 다룰 수 있도록 도와줍니다. string 클래스는 C++ 표준 라이브러리에 포함되어 있으며, #include <string> 헤더 파일을 통해 사용할 수 있습니다.

기존에 문자열 다루는법과 차이 예시

## 기존
```cpp
char str1[20] = "Hello";
char str2[20] = "World";
char str3[40];
strcpy(str3, str1); // str1의 내용을 str3에 복사
strcat(str3, str2); // str2의 내용을 str3에 이어서 복사
cout << str3; // HelloWorld 출력
```

## string 클래스 사용
```cpp
#include <iostream>
#include <string>
using namespace std;

int main() {
    string str1 = "Hello";
    string str2 = "World";
    string str3 = str1 + str2; // 문자열 연결
    cout << str3; // HelloWorld 출력
    return 0;
}
```
## 차이점
기존 방식에서는 문자열을 다루기 위해 char 배열을 선언하고, 문자열 함수를 사용하여 문자열을 복사하거나 연결해야 했습니다. 이 과정에서 메모리 관리와 버퍼 오버플로우 등의 문제가 발생할 수 있었습니다. 반면에 string 클래스는 이러한 문제를 해결하기 위해 동적 메모리를 사용하여 문자열을 관리하며, 다양한 연산을 지원하여 문자열을 쉽게 다룰 수 있도록 도와줍니다. 또한, string 클래스는 C++의 객체 지향적인 특성을 활용하여 문자열을 보다 직관적으로 다룰 수 있게 해줍니다.

비전공자를 위한 포인터 설명 
포인터는 컴퓨터 프로그래밍에서 변수의 메모리 주소를 저장하는 변수입니다. 
포인터를 사용하면 변수의 값을 직접 조작할 수 있으며, 
동적 메모리 할당, 배열, 함수 인자 전달 등 다양한 용도로 활용됩니다.
포인터는 변수의 주소를 저장하기 때문에, 포인터를 통해 변수의 값을 간접적으로 조작할 수 있습니다. 
포인터를 사용하면 메모리 효율성을 높일 수 있으며, 함수 간에 데이터를 전달할 때도 유용합니다. 
포인터는 C++에서 중요한 개념 중 하나이며, 올바르게 사용하면 프로그램의 성능과 유연성을 향상시킬 수 있습니다. 하지만 포인터를 잘못 사용하면 메모리 누수, 버퍼 오버플로우 등의 문제가 발생할 수 있으므로 주의가 필요합니다.


## 함수 구조와 함수 사용법

#### 참고 코드

#include <iostream>
using namespace std;

int add (int a, int b); // 함수 선언

int main() {
    int num1 = 5;
    int num2 = 10;
    int result = add(num1, num2); // 함수 호출
    cout << "Result: " << result << endl; // Result: 15 출력
    return 0;
}
int add(int a, int b) {
    return a + b;
}
#### 함수 구조

1. 반환형: void, int, char, string 등등이 올 수 있다. void는 반환값이 없는 함수라는 뜻이다.
2. 함수 이름: 함수의 이름을 지정한다. 예시에서는 my_strcat이 함수 이름이다.
3. 매개변수: 함수에 전달할 값들을 지정한다. 예시에서는 const string str1, const string str2, string str3가 매개변수이다.
4. 함수 본문: 함수가 수행할 작업을 정의하는 부분이다. 예시에서는 str1과 str2를 str3에 이어붙이는 작업이 수행된다.

#### 반환형에 대하여

각각의 반환형에 따라 함수의 반환값이 달라진다. 예시에서는 int형 반환형을 사용하여 두 정수의 합을 반환하는 함수를 정의하였다.


반환형이 void인 경우 함수는 값을 반환하지 않는다 

예시
'''cpp
#include <iostream>
using namespace std;

void printMessage() {
    cout << "Hello, World!" << endl;
}
int main() {
    printMessage(); // Hello, World! 출력
    return 0;
}
'''

## 필요한 함수를 쉽게 만들기위한 사고방식

1. 문제를 이해한다: 해결하려는 문제를 명확하게 이해하는 것이 중요하다. 문제의 요구사항과 제약 조건을 파악하여 어떤 기능이 필요한지 결정한다.
2. 함수의 목적을 정의한다: 함수를 만들기 전에 함수가 어떤 작업을 수행해야 하는지 명확하게 정의한다. 함수의 입력과 출력, 그리고 함수가 수행할 작업을 구체적으로 생각한다.
3. 함수의 매개변수를 결정한다: 함수가 필요한 데이터를 전달받을 수 있도록 매개변수를 결정한다. 매개변수의 타입과 개수를 고려하여 함수의 인터페이스를 설계한다.
4. 함수의 반환값을 결정한다: 함수가 수행한 작업의 결과를 반환할 필요가 있는지 결정한다. 반환값이 필요한 경우, 반환값의 타입과 의미를 명확하게 정의한다.
5. 함수의 구현을 작성한다: 함수의 본문을 작성하여 함수가 수행할 작업을 구현한다. 함수의 구현은 문제를 해결하는 알고리즘을 기반으로 작성되어야 한다.
6. 함수를 테스트한다: 작성한 함수가 올바르게 작동하는지 테스트한다. 다양한 입력값을 사용하여 함수의 동작을 검증하고, 예상한 결과가 나오는지 확인한다.
7. 함수를 개선한다: 함수가 올바르게 작동하지만 더 효율적이거나 가독성이 좋게 개선할 수 있는 부분이 있는지 검토한다. 필요에 따라 함수의 구현을 최적화하거나, 매개변수와 반환값을 조정할 수 있다.
